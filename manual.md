# POSレジ 取扱説明書

**バージョン**: 329（2026/06/11 時点）
**対応端末**: iPad 第6世代 Safari / 一般的なパソコン・タブレットのブラウザ
**アプリURL**: https://kuchunomura.github.io/pos-app-test

> この説明書は、**パソコンやレジに詳しくない方でも読めるように**、専門用語をできるだけ使わずに書いています。
> 上から順番に読めば、ひととおりの操作ができるようになります。困ったら最後の「[よくある操作 Q&A](#15-よくある操作-qa)」を見てください。

---

## このアプリは何をするもの？

> **タブレットやパソコンの画面で、お会計（レジ打ち）をするためのアプリです。**
> 売ったものとお金は、自動的に **Googleのオンライン表（スプレッドシート）** に記録されていきます。あとから「今日いくら売れたか」を集計したり、グラフで見たりできます。

特別な機械（レジスター）ではなく、**インターネットの画面（ウェブページ）**として動きます。アプリURLをブラウザ（SafariやChrome）で開くだけで使えます。

## 1. 基本画面構成（4つのタブ）

画面のいちばん上に、4つの「タブ（切り替えボタン）」が並んでいます。タップすると画面が切り替わります。

| タブ | 何をする場所か |
|------|---------------|
| 🛒 レジ | **メインの画面。** 商品を選んでお会計をする |
| ⏱ 時間 | アクティビティ（空中ウォーク等）の終了時刻を計算する |
| 📊 売上 | 売れたものの一覧・集計を見る／スプレッドシートに送る |
| ⚙️ 設定 | 商品・カテゴリー・各種設定を変える（普段はあまり触らない） |

> いちばん上のバーには、**今のアプリのバージョン番号**（例：`v329`）と更新日も表示されています。右端の 🔄 ボタンは画面の再読み込み（リロード）ボタンです。

## 2. レジタブ① 商品を選んでカートに入れる

レジタブは画面が **左右2つ**に分かれています。

- **左側** = 商品がたくさん並んでいるエリア（ここから選ぶ）
- **右側** = カート（選んだ商品が入っていく、レシートのような場所）

### 2-1. カテゴリーで絞り込む

左上に、横スクロールできる**カテゴリーのボタン**が並んでいます。

- **すべて**：全部の商品を表示
- カテゴリー名をタップ：そのカテゴリーの商品だけを表示

カテゴリーの例：空中ウォーク／ツリーハウス昼／BBQ／宿泊：ツリーハウス／宿泊：ドーム／宿泊：空中テント／宿泊：ハンモック／備品レンタル／ペット／受付飲食／グッズ／無人販売／消耗品／ピクニック飲食／飲み物／食べ物／その他 など。

### 2-2. 商品をタップしてカートに入れる

**商品のボタンをタップするだけ**で、右側のカートに追加されます。同じ商品をもう一度タップすると数が増えます。

> 宿泊商品（ツリーハウス・ドーム・テント・ハンモック）をタップしたときは、**人数を選ぶ画面**が出ます（→ [6. 宿泊商品の操作](#6-宿泊商品の操作人数選択)）。

### 2-3. 終了時刻の自動表示（空中ウォーク・ツリーハウス昼）

空中ウォーク・ツリーハウス昼のカテゴリーの見出しには、**「今からこのコースを始めたら何時に終わるか」**が自動で表示されます。

- 例：`55分→14:35`　`115分→15:35`　`1DAY→17:00`
- いまカートに入っているコースは**赤色の太字**で目立つようになります。

### 2-4. 画面上の「▼メニュー」ボタン

カテゴリーボタンの右端に、小さな **「▼」ボタン**があります。タップすると、ふだんは隠れている4つのボタンが開きます（もう一度押すと閉じます）。

| ボタン | 使うとき |
|--------|---------|
| 🌙 翌日 | 売上を**翌日の日付**で記録したいとき（→ [8章](#8-翌日翌々日計上モード)） |
| 🌙🌙 翌々日 | 売上を**翌々日の日付**で記録したいとき |
| 📄 請求書 | **請求書・見積書**を作る「請求書モード」に切り替える（→ [9章](#9-請求書見積書の作成)） |
| ✏️ 編集 | 商品ボタンの並べ替え・編集をする「編集モード」に切り替える |

> ふだんの会計では、この中のボタンは使いません。必要なときだけ開いてください。

### 2-5. 金額を手で入力して追加（未登録商品）

左側のいちばん下に金額の入力欄があります。**金額を入力 → 「追加」**を押すと、商品リストに無いものを「未登録商品」としてカートに入れられます。

## 3. レジタブ② カート（右側）の使い方

右側がカート（お会計する商品のリスト）です。

### 3-1. 上部の表示

- **時計**：現在時刻がずっと表示されます。
- **天気アイコン（🌤）**：タップすると今日の天気を選べます（売上データに記録されます）。
- **保留タブ（数字の並び）**：会計を一時とっておく機能（→ [7章](#7-保留機能会計を一時とっておく)）。中身が入っているとオレンジの点が付きます。
- **前日繰越バッジ（黄色）**：前日から「繰越」として記録された売上があると表示されます。

### 3-2. カートの中の操作

| やりたいこと | 操作 |
|-------------|------|
| 商品の数を増やす | カート内の **＋** ボタン |
| 商品の数を減らす | カート内の **－** ボタン（0になると消える） |
| 商品を消す | 右端の **×** |
| カートを全部空にする | 下の **🗑 クリア** ボタン |

### 3-3. 合計金額

カートの下に**合計金額**が大きく出ます。割引が効いているときは、もとの金額に取り消し線が付きます。

## 4. 割引・予約サイトの使い方

左側の下のほうにある **「割引・予約サイト」**の文字をタップすると、割引ボタンが開きます（もう一度押すと閉じます）。

### 4-1. 割引ボタン（押すと赤紫色になる）

| ボタン | 内容 |
|--------|------|
| 村内（半額） | レンタル以外の商品が半額になる。一部の商品（BBQなど）は無料になる |
| 団体割引 | 団体向けの割引。割引率（◯%）は設定で変えられます（ボタンに `-10%` のように表示） |
| 無料割引 | 対象商品を無料（¥0）にする |
| 宿泊者 | 対象商品を宿泊者向けの割引価格にする |

ボタンを押すと割引が効き、もう一度押すと解除されます。

### 4-2. 予約サイトボタン（押すと青色になる）

楽天・じゃらん・SOU・年間パス（昼）・年間パス（夜）などがあります。
これを選ぶと、宿泊商品に **「支払済」のバッジ**が付き、料金が **¥0** として扱われます（=その予約サイト経由で前払い済み、という意味）。

> **ポイント**：割引と予約サイトは、それぞれ選べます。同じボタンをもう一度押すと解除できます。
> 割引・予約サイトのボタンの追加や削除は、[設定タブ](#12-設定タブ設定)でできます（村内・宿泊者は固定で消せません）。

### 4-3. 属性入力（👤）

割引バーの近くにある **👤** をタップすると、お客さんの年齢層・国籍を選べます（あとで集計に使えます）。複数選べて、会計が終わるとリセットされます。

## 5. 支払い（お会計）のしかた

カートに商品を入れたら、お会計に進みます。**進み方は2通り**ありますが、おすすめは「決済へ」ボタンです。

### 5-1.【おすすめ】「決済へ」ボタンで確認してから会計

1. カート下の合計金額の右にある **「決済へ」**ボタンを押す
2. **お会計確認の画面**が開く（買うものの一覧と合計金額がもう一度出る）
3. 内容を確認したら、支払方法のボタンを押す：
   - **💴 現金**
   - **💳 カード**
   - **📱 iD/QP/交通系**（電子マネー）

> この確認画面には注意書きが出ます：**※クレジットは物理カードのみ（タッチ決済は不可）　※PayPayは使えません**

### 5-2. 支払方法のボタンを直接押す

カート下にも **💴現金 / 💳カード / 📱iD/QP/交通系** のボタンが並んでいます。急いでいるときはここを直接押してもOKです。

### 5-3. 現金払いの操作

1. **💴 現金**を押す
2. 確認画面が開く
3. 「受取金額」欄に、お客さんから受け取った金額を入力
   - 金額ボタン（50／100／500／1,000／5,000／10,000）を押すと足し算で入力できます
   - **C** … 入力を消す
   - **ちょうど** … 合計と同じ金額を自動で入れる
4. **お釣りが自動で計算**されて表示されます
5. **決済完了**を押す

### 5-4. カード・電子マネーの操作

1. **💳 カード** または **📱 iD/QP/交通系**を押す
2. 確認画面（金額の確認）が出る
3. **決済完了**を押す

### 5-5. 決済が終わったあとの画面

会計が終わると完了画面が出ます。

| やること | 動作 |
|---------|------|
| 画面の外側（暗い部分）をタップ／✅次のお客様へ | カートを空にして次の会計へ |
| 🔴 印をつけて閉じる | 「あとで確認が必要」な売上に赤い印を付けて閉じる |

> 決済が終わると、売上は**自動でGoogleスプレッドシートに送られます**（手動操作は不要）。送信中・送信済みの状態は売上タブで確認できます（→ [11章](#11-売上タブ売上)）。

## 6. 宿泊商品の操作（人数選択）

ツリーハウス・ドーム・テント・ハンモックの商品をタップすると、**人数を選ぶ画面**が開きます。
人数によって1人あたりの料金が変わります。

例（ツリーハウス）：

| 人数 | 1棟の料金 |
|------|----------|
| 1名 | ¥14,000 |
| 2名 | ¥12,000 |
| 3名 | ¥10,000 |
| 4名 | ¥8,000 |
| 5名以上 | ¥8,000 |

> ドームやテントには色の点（●）が付いていて、種類（ミラー・クリア・グレーなど）が見分けられます。

### 予約サイト経由の「支払済」

割引バーで**楽天・じゃらん・SOU・年間パス**を選ぶと、宿泊商品に「支払済」バッジが付いて料金が¥0になります（バッジの文字や色は設定で変えられます）。

## 7. 保留機能（会計を一時とっておく）

複数のお客さんの会計を**同時に進めたい**ときに使います。最大10件までとっておけます。

### とっておく（保留する）

1. カートに商品が入った状態で **⏸ 保留**ボタンを押す
2. タイトルを入力する画面が出る（空欄なら時刻がタイトルになる）
3. **保留する**を押すと、カートが空になり、保留タブに保存される

### 戻す

1. カート上部の**保留タブ（数字）**を押す
2. そのときのカートが戻ってくる

> 中身が入っている保留タブには**オレンジの点（●）**が付きます。

## 8. 翌日・翌々日計上モード

チェックインは今日だけど、**料金は翌日（または翌々日）の売上として記録したい**ときに使います。

1. 画面上の **「▼メニュー」**を開き、**🌙 翌日** または **🌙🌙 翌々日**を押す
2. カートの上に黄色／茶色のバナー「🌙 翌日計上モード ON」が出る
3. この状態で会計すると、**売上の日付が翌日（翌々日）**で記録される
4. もう一度同じボタンを押すと解除

> 売上タブの絞り込みで「明日」「明後日」を選ぶと、翌日計上した売上を確認できます。

## 9. 請求書・見積書の作成

お客さんに渡す**請求書や見積書**を、スプレッドシート上に帳票（書類）の形で作れます。作り方は2通りあります。

### 9-1. カート（今の会計内容）から作る

1. 画面上の **「▼メニュー」**を開き、**📄 請求書**を押す（ボタンの色が変わり「請求書モード」になる）
2. すると、カート下の支払ボタンが **「📄 請求書・見積書を作成」**に変わる
3. 商品をカートに入れた状態でそのボタンを押す
4. 作成画面で次を選ぶ：
   - **種別**：請求書 か 見積書
   - **売上計上**：
     - 「今日の日付で売上計上」… 今日の売上としても記録する
     - 「別日で売上計上」… 日付を指定して、その日の売上として記録する
     - 「売上計上しない」… 書類だけ作る（見積りなどで、まだ売上にしないとき）
   - **宛先**（任意）：例「田中様」「〇〇グループ」
   - **備考**（任意）
5. **作成する**を押すと、スプレッドシートに請求書／見積書のシートが作られます

> 作り終わると請求書モードは自動で解除されます。もう一度ふつうの会計に戻ります。

### 9-2. 過去の売上から作る

売上タブの明細から特定の売上を開いて、そこから請求書・見積書を作ることもできます（宛先・備考を入れて作成）。

## 10. 時間計算タブ（⏱時間）

アクティビティが**何時に終わるか**をすばやく計算する道具です。会計とは関係なく使えます。

### 単体で計算

1. **施設**（空中ウォーク／ツリーハウス昼）を選ぶ
2. **コース**（55分／115分／1DAY）を選ぶ
3. **開始時間**を入れる（**今すぐ**ボタンで現在時刻が入る）
4. 終了時刻と残り時間が表示される

| 施設 | 営業時間 | 最終受付 |
|------|---------|---------|
| 空中ウォーク | 9:00〜17:00 | 16:00 |
| ツリーハウス昼 | 10:00〜16:00 | 15:00 |

### 両方つづけて利用する場合（コンボ計算）

1. **順番**（空中ウォーク→ツリーハウス、またはその逆）を選ぶ
2. それぞれのコース時間を選ぶ
3. 開始時間を入れる
4. 各施設の終了時刻が出る

## 11. 売上タブ（📊売上）

売れたものの一覧や集計を見る画面です。

### 11-1. 明細ビュー（📋 明細）— 一覧で見る

| やりたいこと | 操作 |
|-------------|------|
| 今日の売上 | **📅 今日** |
| 昨日の売上 | **昨日** |
| 日付で絞る | 日付欄に範囲を入れて **絞込** |
| 全部見る | **全件** |

上部には、売上合計・件数・人数・客単価のまとめが出ます。

**一覧の各行の見方：**

- いちばん左の **🔴／⚪** … 「要確認」の印。タップで付けたり外したりできる
- 日時の下に **📱端末名**（どの端末からの売上か）
- 日時の下の**同期バッジ**でスプレッドシートへの送信状況がわかります：
  - 🟢 **同期済** … 送信できている
  - 🔴 **要再送** ＋ **再送ボタン** … 送信に失敗。**再送**ボタンで送り直せる
  - **送信中...** … いま送っている途中
- 🌙マーク … 翌日計上した売上
- 📝メモ … メモが付いている売上

### 11-2. 売上の修正・削除

- 行をタップ → **詳細画面**が開く。メモ・属性の変更、🔴印の付け外し、削除ができる
- 行の **「カートで修正」**ボタン → その売上がカートに戻る。直してもう一度会計すると、売上が更新される

### 11-3. 集計ビュー（📊 集計）— グラフで見る

期間（今日／今週／今月／カスタム）を選んで、いろいろなグラフを見られます。

| グラフ | 内容 |
|-------|------|
| 日別 | 日ごとの売上 |
| 時間帯別 | 時間帯ごとの件数 |
| 曜日別 | 曜日ごとの集計 |
| 商品別 | 商品ごとの売上・数量 |
| 属性別 | 年齢層・国籍ごとの組数・人数 |
| 割引別 | 割引種別ごとの件数 |

月ごとの集計をまとめて見る「月別集計」もあります。

### 11-4. CSV出力

**📥 CSV** を押すと、今絞り込んでいる売上をCSVファイル（表計算ソフトで開けるファイル）に書き出せます。

### 11-5. スプレッドシートへの同期（📊 同期）

会計のたびに自動で送られますが、**手動でまとめて送る**こともできます。**📊 同期 ▾** を押すとメニューが出ます。

| メニュー | 内容 |
|---------|------|
| 今日分 | 今日の売上を送る |
| 昨日分 | 昨日の売上を送る |
| 絞込分 | いま絞り込んでいる分を送る |
| 全件 | すべての売上を送る |

> ⚠️ **「全件」は要注意**：シートをいったん全部消してから送り直します。複数の端末で使っている場合、ある端末で「全件」をすると、**その端末が知らない他の端末の売上がシートから消えてしまいます**。複数台のときは「今日分」「昨日分」を使ってください（→ [14章](#14-複数の端末で使うときの注意)）。

### 11-6. 月初シート切替（📂）— 月が替わったとき

毎月、新しいGoogleスプレッドシートに記録先を切り替える運用をしている場合に使います。

1. **📂 月初シート切替**ボタンを押す
2. 新しく作ったスプレッドシートのURL（またはID）を貼り付ける
   - 「同期先シート（GAS）」… 売上を記録するシート
   - 「手動集計シート」… 手動集計用のシート（変えないなら空欄でOK）
3. **切り替える**を押す

> これ以降の売上は、新しいスプレッドシートに記録されます。

## 13. Googleスプレッドシート連携の仕組み

このアプリは、売上を **Google Apps Script（GAS）** という仕組みを通じて、Googleスプレッドシートに自動で記録します。

### 13-1. 自動で送られるタイミング

**会計が完了するたびに自動で送られます。** 手動操作は不要です。

| できごと | スプレッドシートでの動き |
|---------|------------------------|
| 会計完了 | 行を追加 |
| 「カートで修正」して再会計 | 元の行を消して新しい行を追加 |
| 詳細画面でメモ・属性を変更 | その行を更新 |
| 詳細画面から削除 | その行を削除 |

> 送信に失敗したときは、売上タブの明細に **🔴要再送**バッジが出ます。**再送**ボタンで送り直せます。

### 13-2. 設定（スプレッドシート連携）

| 項目 | 説明 |
|------|------|
| GAS WebアプリURL | 送信先となるGASのURL |
| 端末名 | スプレッドシートに記録される端末の名前（例：レジ1） |

**接続テスト**：URLを保存してから「🔗 接続テスト」を押すと、ちゃんとつながるか確認できます。

> このTEST版アプリは、GASのURLが内部で固定されています（設定で変えても起動時に元に戻ります）。本番（main版）では自由に変更できます。

## 14. 複数の端末で使うときの注意

### データはどこにある？

このアプリの売上・設定は、**それぞれの端末のブラウザの中**に保存されます（localStorageといいます）。**端末どうしでデータは自動共有されません。**

ただし、売上は会計のたびにスプレッドシートへ送られるので、**スプレッドシートには全部の端末の売上が集まります。**

```
端末A の売上  ┐
               ├→ Googleスプレッドシート（全部まとまる）
端末B の売上  ┘
```

### 端末名は必ず設定する

どの端末からの売上か見分けられるように、設定タブの「スプレッドシート連携」で**端末名（例：レジ1、レジ2）**を必ず設定してください。

### 「全件同期」の落とし穴（重要）

ある端末で「全件」同期をすると、シートをいったん空にして送り直すため、**その端末が知らない他端末の売上が消えます。** 複数台で使うときは「今日分」「昨日分」などの部分同期を使ってください。

## 15. よくある操作 Q&A

**Q. 会計を間違えた金額でやってしまった**
A. 📊売上タブの明細でその行の **「カートで修正」**を押し、直してもう一度会計してください。

**Q. 何人ものお客さんを同時に会計したい**
A. **⏸ 保留**機能を使います。1組目を保留 → 2組目を入力 → 保留タブで切り替えて精算。

**Q. 天気を記録したい**
A. カート右上の天気アイコン（🌤）をタップして選ぶと、その日の売上に記録されます。

**Q. 請求書／見積書を作りたい**
A. 「▼メニュー」→ 📄請求書 で請求書モードにして、カートから作成します（→ [9章](#9-請求書見積書の作成)）。

**Q. 別のお客さんの会計に進みたい**
A. 決済完了画面で「次のお客様へ」を押すか、画面の外側をタップするとカートが空になります。

**Q. スプレッドシートに売上が反映されない**
A. ①設定タブ → スプレッドシート連携 → 「🔗 接続テスト」で「✅ GAS接続OK」が出るか確認。②OKなら 📊同期 →「今日分」で送り直す。③明細に🔴要再送バッジがあれば「再送」ボタンを押す。

**Q. 別の端末から見ると売上が無い**
A. 端末ごとにデータは別です。売上はスプレッドシートには集まっているので、そちらで全体を確認できます。端末の画面で見たいときは、その端末のデータをエクスポート→こちらにインポートしてください。

**Q. 月が替わったので記録先を新しいシートにしたい**
A. 売上タブの「📂 月初シート切替」で新しいスプレッドシートのURLを設定します（→ [11-6](#11-6-月初シート切替-月が替わったとき)）。

**Q. アプリを開いたら古い画面のままだった**
A. ブラウザに古いデータが残っています。[16章](#16-アプリの更新画面が古いときの対処)を参照してください。

## 16. アプリの更新・画面が古いときの対処

### バージョンの確認

画面いちばん上のバー、または設定タブの最上部に、今のバージョンが出ます。

```
POSレジ ver.329
更新: 2026/06/11 | TEST版
```

新しいバージョンが出ているとき、アプリが自動で気づいて「新しいバージョンがあります。会計完了後に🔄ボタンで更新してください」と案内することがあります。その場合は会計を終えてから右上の **🔄** を押してください。（カートが空のときは自動で更新されることもあります。）

### 画面が古いまま変わらないとき（キャッシュの削除）

> ⚠️ **iPadのSafariで「Webサイトデータを削除」すると、売上・設定も全部消えます。** 先に必ず[エクスポート（バックアップ）](#17-データのバックアップと引き継ぎ)してください。

##### iPad（Safari）
1. 「設定」アプリ → 「Safari」→「詳細」→「Webサイトデータ」
2. 「すべてのWebサイトデータを削除」→ アプリを開き直す
3. （データを消したくない場合）プライベートタブでアプリURLを開くと、最新版を確認だけできます

##### パソコン（Chrome / Edge）
- **`Ctrl + Shift + R`**（Macは `Cmd + Shift + R`）を押す。これで最新版になります。**売上・設定は消えません。**

##### Androidタブレット（Chrome）
1. Chrome右上「︙」→「設定」→「プライバシーとセキュリティ」→「閲覧履歴データの削除」
2. **「キャッシュされた画像とファイル」だけ**にチェック（「Cookieとサイトデータ」はチェックしない）→ 削除
3. これなら売上・設定は消えません

## 17. データのバックアップと引き継ぎ

### エクスポート（バックアップを取る）

設定タブの **「📤 全データをエクスポート」**を押すと、商品・設定・売上などをファイル1つに書き出せます。**週1回くらいは取っておくと安心です。**

### インポート（別の端末に引き継ぐ・復元する）

**「📥 インポート」**で、書き出したファイルを読み込みます。
- 売上・保留は**追加**（重複しない）
- 設定（商品・GAS URLなど）は**上書き**

### データリセット

特定の項目を最初の状態（初期値）に戻せます。**手動で変えた設定は元に戻せません。**（売上データとGAS URLは消えません。）

## 18. スプレッドシートの中身（参考）

売上は「M/D売上」（例：6/11売上）というシート名で、日付ごとに記録されます。

| 列 | 内容 |
|----|------|
| A | 日時 |
| B | 売上合計 |
| C | 商品名 |
| D | カテゴリー |
| E | 数量 |
| F | 単価 |
| G | 小計 |
| H | 人数 |
| I | 割引 |
| J | 支払方法 |
| K | 年齢層 |
| L | 国籍 |
| M | 天気 |
| N | メモ |
| O | 売上ID |
| P | 端末名 |

シートの1行目には集計（件数・売上・現金/カード/電子の合計）、2行目にはレジ現金の手入力欄と差額計算が自動で入ります。右側には商品別・年齢層別・国籍別の集計表が自動で作られます。
請求書・見積書は別のシート（帳票の形）として作られます。
