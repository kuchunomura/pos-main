1. このシステムの全体像

一言で言うと

タブレット（iPad・PCでも）で動くレジアプリ。売上データはGoogleスプレッドシートに自動で記録される。

システムを構成する3つのパーツ

2. 使う「道具」の説明

改修作業で使う道具は4つあります。それぞれ全然別のものです。

道具① VS Code（青いアプリ）

役割：ファイルを開いて編集するためのソフト

Wordで文書を編集するのと同じように、VS CodeはHTMLやJavaScriptのファイルを編集するためのソフトです。プログラマー向けの高機能なテキストエディタです。

アイコン: 青い「< >」のマーク
正式名称: Visual Studio Code（マイクロソフト製）
用途: index.html や gas_script.js を開いて中身を確認・編集する

VS Code自体に改修の能力はありません。ファイルを「開く・見る・編集する」だけのソフトです。

道具② Claude Code（VS Codeの中にある青いチャット）

役割：AIにファイルの改修を依頼するチャット画面

VS Codeの中に組み込まれているAIアシスタントです。チャットで日本語で指示すると、AIがファイルを自動で書き換えてくれます。

どこにある: VS Codeの左側のバーにあるアイコン、またはパネル内
見た目: チャット画面（メッセージを入力して送信する）
用途: 「○○を追加してください」と日本語で頼むと、
       AIが index.html などのファイルを自動で編集してくれる

重要: Claude Code は「頭脳」であり、VS Code は「作業台」です。
Claude Code が VS Code のファイルを直接書き換えます。

道具③ ターミナル（黒い画面）

役割：コンピューターに命令を文字で入力する画面

ターミナルとは「黒い画面に文字でコマンド（命令）を入力する場所」です。マウスで操作する代わりに、キーボードだけでコンピューターに指示を出します。

見た目: 黒または暗い背景に白い文字
VS Codeの中にある: 上メニュー「Terminal」→「New Terminal」で開く
用途: GitHub へのアップロード（git コマンド）など

普段はClaude Code が代わりにターミナルを操作してくれるので、自分で文字を入力することはほぼありません。

道具④ GitHub（インターネット上のファイル置き場）

役割：ファイルを保存・公開するクラウドサービス

GitHubはプログラムのファイルを保存・共有するためのサービスです。Googleドライブと似ていますが、プログラマー向けの特別な機能（変更履歴の管理など）があります。

種類1: GitHub（ウェブサイト） → github.com にある
種類2: GitHub Pages → GitHub上のファイルをウェブサイトとして公開する機能

このプロジェクトでは：

  • ファイル置き場として → index.html などを保存
  • ウェブサイト公開として → タブレットがアクセスするURLになる

5. アプリのデータの流れ

データの保存場所は2か所ある

【端末内】 localStorage（ローカルストレージ）
  ↓ 端末のブラウザ内に保存される
  ・商品リスト
  ・カテゴリー設定
  ・売上履歴
  ・保留データ
  ・GAS URL等の各種設定
  ※ インターネット不要で動く
  ※ ブラウザのデータを消すと消える！

【Google】 Googleスプレッドシート
  ↓ GASを通じてインターネット経由で保存
  ・売上データ（日付別シート）
  ※ インターネット必要
  ※ ブラウザのデータを消しても残る

決済したときのデータの流れ

1. お客様の商品をタップ → カートに追加（端末内）

2. 「現金」「カード」「電子」ボタンをタップ

3. 売上データが端末内のlocalStorageに保存される

4. 同時に、インターネット経由でGASにデータを送信

5. GASがスプレッドシートに新しい行を書き込む

6. スプレッドシートの集計が自動更新される

複数台で使うとき

iPad（レジ1）  →  端末内データ A  →┐
                                    ├→ GAS → スプレッドシート（全台合算）
iPad（レジ2）  →  端末内データ B  →┘

※ 端末どうしは直接つながっていない
※ スプレッドシートが全員の集計場所

6. 改修の基本手順

全体の流れ（必ずこの順番で）

Step 1: Claude Code にチャットで依頼する
         「test版のindex.htmlに○○を追加してください」

         ↓

Step 2: Claude Code がファイルを自動で書き換える
         （パソコン内のファイルが変わるだけ。まだネットには反映されない）

         ↓

Step 3: ブラウザでtest版URLを開いて確認する
         https://kuchunomura.github.io/pos-app-test
         ※ この時点ではまだ古いバージョンが表示される可能性あり
         ※ まずGitHubにアップロードしてから確認する

         ↓

Step 4: GitHubにアップロード（デプロイ）する
         Claude Codeに「GitHubにアップロードしてください」と頼む

         ↓

Step 5: 数分待ってURLを再読み込みして確認
         変更が反映されているか確認する

         ↓

Step 6: 問題なければmain版にも同じ変更を適用
         「main版のindex.htmlにも同じ変更をしてください」
         → 再度GitHubにアップロード

Claude Code への頼み方（コツ）

良い頼み方の例

「test版のindex.htmlに以下の変更をしてください。
 カテゴリーに『テイクアウト』を追加して、
 商品は『コーヒー（400円）』『紅茶（350円）』
 の2つを登録してください。
 カテゴリーの色は背景が薄い青、枠が青、文字が紺にしてください。」

悪い頼み方の例

「コーヒーを追加して」
  → どのファイル？何円？どのカテゴリー？情報が足りない

「全部直して」
  → 何を直すのか分からない

必ず伝えるべき情報

  • どのファイルを変更するか（test版 or main版）
  • 何を変えたいか（追加・削除・変更・デザイン変更）
  • 具体的な内容（商品名・金額・色など）

7. 「test版」と「main版」の使い分け

なぜ2つあるのか

改修は必ず「失敗するリスク」があります。もし本番（main版）で直接作業して壊れたら、営業中にレジが使えなくなります。それを防ぐために「まずtest版で試す」という手順があります。

test版 = 練習場・実験場
main版 = 本番・実際に使う場所

test版（pos test/index.html）

  • URL: https://kuchunomura.github.io/pos-app-test
  • 改修のお試し・動作確認用
  • ここで壊れても本番には影響しない
  • 改修はまずtest版で行う

main版（pos main/index.html）

  • 実際のレジとして使用中
  • ここを壊すと営業に影響が出る
  • test版で確認後にmain版に反映する

test版 → main版への反映手順

test版で確認できたら、Claude Codeにこう頼む：

「test版のindex.htmlに加えた○○の変更を、
 main版のindex.htmlにも同じように適用してください。」

8. デプロイとは何か

「デプロイ」をかんたんに説明すると

パソコン内で編集したファイルを、インターネット上に公開すること

【パソコンの中】                    【インターネット上】
  pos main/                          GitHub Pages
    index.html   ─── デプロイ ───→    URL（タブレットがアクセスする場所）

なぜデプロイが必要か

ファイルを編集しただけでは、それはパソコンの中にしかありません。タブレットからURLを開いてもまだ古いバージョンが表示されます。GitHubにアップロード（＝デプロイ）して初めてタブレットに反映されます。

ファイルを編集 → パソコンの中だけ変わる
デプロイする  → インターネット上が更新される → タブレットに反映

デプロイの仕組み（Git）

GitHubへのアップロードには「Git（ギット）」というシステムを使います。難しく考える必要はありません。Claude Codeが全部やってくれます。

Git の主な操作:
  git add     → 「このファイルをアップロード対象にする」
  git commit  → 「変更内容に名前をつけて記録する」
  git push    → 「GitHubにアップロードする」

※ この3つをまとめて「デプロイ」と呼んでいる
※ Claude Code に頼めばすべて自動でやってくれる

バージョン番号のルール（重要）

index.html を変更したら、画面に出るバージョン番号を必ず1つ増やす決まりになっています（どんなに小さな修正でも）。これは「今タブレットに表示されているのが最新版かどうか」を見分けるための番号です。

変更のたびに、セットで更新する3つ:
  ・APP_VERSION      → バージョン番号を +1（例: 329 → 330）
  ・APP_BUILD_DATE   → 更新した日時に書き換える
  ・CLAUDE.md の「現在のバージョン」 → 同じ番号・日付に揃える

これらは Claude Code に頼めば自動でやってくれます（「バージョンを上げてデプロイして」と頼むだけ）。番号を上げ忘れると、タブレット側が「新しい版が出た」と気づけず、古い画面のままになってしまいます。

デプロイの頼み方

Claude Codeへ:
「test版をGitHubにアップロードしてください。
 コメント（コミットメッセージ）は『テイクアウトカテゴリーを追加』にしてください。」

Claude Codeがターミナルで以下を自動実行します（確認のポップアップが出たらOKを押す）：

git add "pos reji/pos test/index.html"
git commit -m "テイクアウトカテゴリーを追加"
git push origin main

デプロイ後の確認

  1. 数分待つ（GitHub Pagesの更新に時間がかかる）
  2. ブラウザでURLを開く
  3. 設定タブのバージョン番号が新しくなっているか確認

古いバージョンが表示される場合 → キャッシュが残っている可能性がある。取扱説明書の「アプリ更新・キャッシュ問題」参照。

9. GAS（Google Apps Script）の改修

GASとは

GAS = Google Apps Script

Googleのサーバーで動くプログラム。
レジから送られてきた売上データをスプレッドシートに書き込む「翻訳係」。

GASを変更する必要があるケース

  • スプレッドシートの列を追加・変更したい
  • 集計の計算方法を変えたい
  • 新しいデータ項目を記録したい

GASの改修手順

  1. gas_script.js を Claude Code で編集する（パソコン内のファイル）
  2. Google Apps Script のサイト（script.google.com）をブラウザで開く
  3. 対象プロジェクトのコードをすべて選択して削除
  4. 編集した gas_script.js の内容をコピーして貼り付ける
  5. 保存する（Ctrl+S）
  6. 「デプロイ」→「デプロイを管理」→「新しいデプロイ」

注意: GASを変更すると スプレッドシートへの書き込みが止まる ことがあります。
改修時は必ず設定タブの「接続テスト」で確認してください。

GASとindex.htmlの連携

index.html（レジ）  →  GAS WebアプリURL  →  gas_script.js  →  スプレッドシート

GAS WebアプリURL = GASの「住所」
設定タブ → スプレッドシート連携 → GAS WebアプリURL に入力されている

11. 困ったときのチェックリスト

アプリが動かない・エラーが出る

  • [ ] Claude Codeに「エラーが出ています」と伝える（何のエラーか教えれば解決してくれる）
  • [ ] Gitの履歴から戻せる → 「1つ前のバージョンに戻してください」と頼む

スプレッドシートに売上が書き込まれない

  • [ ] 設定タブ → スプレッドシート連携 → 「接続テスト」でOKが出るか確認
  • [ ] GAS WebアプリURLが正しく設定されているか確認
  • [ ] 売上タブ → 「同期」→「今日分」で手動送信を試みる

改修後にアプリが古いままに見える

  • [ ] GitHubにアップロード（デプロイ）したか確認
  • [ ] ブラウザのキャッシュをクリアしたか確認（取扱説明書 12章参照）
  • [ ] 設定タブのバージョン番号が新しくなったか確認

誤った改修でデータが消えた・壊れた

  • [ ] Gitの履歴から古いファイルを復元できる（「1つ前に戻してください」と頼む）
  • [ ] localStorage内のデータが消えた場合 → エクスポートしたJSONファイルからインポートで復元
  • [ ] スプレッドシートのデータは消えない（GASでのみ変更される）

12. 重要な用語集

用語意味
VS Codeファイルを編集するソフト（青いアイコン）。Wordのプログラマー版
Claude CodeVS Codeの中にあるAIチャット。日本語で頼むとファイルを自動編集してくれる
ターミナル黒い画面。文字でコンピューターに命令する場所。Claude Codeが自動で使う
GitHubプログラムのファイルをインターネット上に保存・公開するサービス
GitHub PagesGitHubのファイルをウェブサイトとして公開する機能。URLでアクセスできる
デプロイパソコン内のファイルをGitHubにアップロードして、URLに反映させること
Gitファイルの変更履歴を管理するシステム。git push でGitHubに送る
git pushGitHubにアップロードするコマンド。ターミナルで実行する
localStorageブラウザ内のデータ保存場所。端末ごとに独立。ブラウザのデータを消すと消える
GASGoogle Apps Script。Googleのサーバーで動くプログラム。スプレッドシートへの書き込み担当
キャッシュブラウザが保存している古いページデータ。クリアすると最新版が表示される
JSONデータをやり取りするときのフォーマット。エクスポート/インポートで使う
index.htmlレジアプリ本体。HTML・CSS・JavaScriptがすべて入った1ファイル
CLAUDE.mdClaude Codeがプロジェクトを理解するための技術仕様書。触らなくてよい
コミットGitでファイルの変更を記録する操作。変更に名前をつけること
SS_IDスプレッドシートの固有ID。gas_script.jsの先頭に書かれている。変更しない
